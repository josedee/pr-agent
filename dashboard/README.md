# 🤖 PR Agent Dashboard

A beautiful, responsive dashboard for visualizing pending pull requests across multiple repositories.

## ✨ Features

- 📊 **Real-time Statistics** - Total PRs, reviewers, priority PRs, and average age
- 🎨 **Dark/Light Mode** - Automatic theme switching with localStorage persistence
- 🔍 **Search & Filter** - Find PRs quickly with instant search
- 👥 **Multiple Views** - View by all PRs, priority, reviewer, or repository
- 📱 **Responsive Design** - Works perfectly on desktop, tablet, and mobile
- ⚡ **Fast Loading** - Static data generation for instant page loads
- 🎯 **Priority Indicators** - Color-coded PR age (low, medium, high, critical)
- 🔄 **Auto-refresh** - Data updates automatically via GitHub Actions

## 🚀 Quick Start

### Option 1: Local Testing

1. **Open the dashboard locally:**
   ```bash
   cd pr-agent-dashboard
   # Open index.html in your browser
   open index.html  # macOS
   # or
   start index.html  # Windows
   # or
   xdg-open index.html  # Linux
   ```

2. **The dashboard will load with sample data** from `data.json`

### Option 2: GitHub Pages Deployment

1. **Enable GitHub Pages:**
   - Go to repository Settings → Pages
   - Source: Deploy from a branch
   - Branch: `gh-pages` / `root`
   - Save

2. **Your dashboard will be available at:**
   ```
   https://YOUR_USERNAME.github.io/pr-agent/
   ```

## 🔧 How It Works

### Data Flow

```
┌─────────────────────────────────────────────────────┐
│  1. GitHub Actions Workflow                         │
│     - Runs on schedule (e.g., hourly)              │
│     - Fetches PRs from repositories                 │
│     - Calls generate-dashboard-data.js              │
└─────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────┐
│  2. Data Generator (generate-dashboard-data.js)     │
│     - Processes PR data                             │
│     - Calculates statistics                         │
│     - Groups by reviewer/repository                 │
│     - Generates data.json                           │
└─────────────────────────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────┐
│  3. GitHub Pages                                     │
│     - Serves static files                           │
│     - index.html loads data.json                    │
│     - Renders beautiful dashboard                   │
└─────────────────────────────────────────────────────┘
```

## 🛠️ Customization

### Changing Colors

Edit `styles.css` to customize colors:

```css
:root {
    --color-primary: #0d6efd;  /* Primary color */
    --priority-low: #28a745;    /* Low priority */
    --priority-medium: #ffc107; /* Medium priority */
    --priority-high: #fd7e14;   /* High priority */
    --priority-critical: #dc3545; /* Critical priority */
}
```

### Changing Priority Thresholds

Edit `generate-dashboard-data.js`:

```javascript
function getPriority(pr) {
  const days = pr.age.days;
  if (days >= 7) return 'critical';  // Change threshold
  if (days >= 3) return 'high';
  if (days >= 1) return 'medium';
  return 'low';
}
```

## 🐛 Troubleshooting

### Dashboard shows "Failed to Load Data"

**Cause**: `data.json` file not found or invalid

**Solutions**:
1. Ensure GitHub Actions workflow has run successfully
2. Check that `data.json` exists in the repository
3. Verify the file is valid JSON
4. Check browser console for detailed error messages

### Dashboard shows old data

**Cause**: Browser cache or workflow hasn't run

**Solutions**:
1. Click the "Refresh" button
2. Hard refresh: Ctrl+Shift+R (Windows/Linux) or Cmd+Shift+R (Mac)
3. Check when the workflow last ran in GitHub Actions
4. Manually trigger the workflow if needed

### Styles not loading correctly

**Cause**: CSS file path issue or browser cache

**Solutions**:
1. Clear browser cache
2. Check that `styles.css` is in the same directory as `index.html`
3. Open browser DevTools and check for 404 errors

### Theme not persisting

**Cause**: localStorage not available or disabled

**Solutions**:
1. Check if browser allows localStorage
2. Ensure you're not in private/incognito mode
3. Check browser console for errors
